## Runtime design

### External references

`lifecycle.js` builds a `Map` for the allow-listed key `valve_actuation_expectations.expectation_id` inside the same transaction used to finalize the entry. The resolver returns only expectations whose `zone_id` equals the resolved plot zone and whose valve device belongs to the authenticated local user and current gateway. A plot without a zone resolves an empty set. The map is passed to create, draft-promotion, and correction validation. Unknown reference metadata remains fail-closed.

### Command ACKs

`classifyAckResult('EXPIRED')` returns `EXPIRED`, making the existing terminal branch reachable. For newly queued generic ACKs, the normalized ACK written to `command_ack_outbox` is also stored as `applied_commands.result_detail`; an exact replay returns that durable value unchanged. Legacy rows retain their existing replay behavior. A defensive payload guard prevents future injected validators from authorizing a dereference of a non-object payload.

### Research package

The package remains streaming and gains `records.ndjson`. Each line is one envelope with `record_type` equal to `entry`, `value`, or `vocab_mapping`, and `data` equal to the corresponding typed row object. CSV members apply the same string-only formula neutralization as the wide export. `records.ndjson` preserves exact source strings and JSON number types, so a consumer can reconstruct every exported row without guessing whether an apostrophe was source data or spreadsheet escaping. The manifest lists and hashes the new member and describes the CSV safety transformation.

### Outbox JSON

All three sync paths use a strict object parser. Invalid JSON, JSON `null`, arrays, and scalar JSON are rejected with the event UUID in the error. Gateway migration aborts and rolls back before updating any outbox row. Normal and forced delivery abort before the HTTP call and therefore cannot mark malformed rows delivered. Existing retry/error reporting remains the visible recovery path.

## Schema and contract design

- `journal_plot_settings.plot_uuid` references `journal_plots(plot_uuid) ON DELETE CASCADE` in migration 0018, the seed, and every bundled database.
- Journal-event schemas require `occurredAt`; non-journal event compatibility is unchanged.
- Command-only resource wrappers override compared UUID fields with `CanonicalUuid`.
- The catalog generator treats a recorded checksum as an immutable recovery constraint, even when the migration file is missing.
- The custom schema harness remains dependency-free for credential-free CI. It validates its supported keyword shapes, counts Unicode code points for string lengths, and uses JSON deep equality for `const`, `enum`, and `uniqueItems`.

## Flow integration

New journal command and HTTP function nodes declare only `osiLib`. Each node loads required helpers once per invocation, validates every loader result, emits a visible error, and returns its existing fail-closed output shape when a helper is unavailable. The loader registry gains explicit `osi-db-helper` and `osi-command-ledger` entries. Existing non-journal flow helper conventions are not redesigned in this PR.
