> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development for independent implementation slices and superpowers:verification-before-completion before each merge/deploy boundary. The orchestrator owns shared-file integration and live actions.

**Goal:** Execute the reviewed repair plans in a safe order, produce green and bisectable commits, repair Kaba100 with one coordinated edge deployment, then harden the deployment boundary in a separate release.

**Architecture:** Treat the findings as two release trains. Repair Train A fixes durable sync delivery, writer/database behavior, ChirpStack reconciliation, and Device API status propagation on the existing deployment path, with a bounded compatibility-set restore so its new helpers can never be paired with an old rolled-back flow. Repair Train B replaces that temporary guard with complete release construction, personalization, rollback, pipeline routing, configuration, and CI; it receives a separate Kaba100 rehearsal only after Train A is stable. The OSI Server contract work remains a separate repository PR. This program permits only its explicitly approved non-production test-server deployment after backup, migration-ledger validation, and local/server CI; it never permits production deployment.
