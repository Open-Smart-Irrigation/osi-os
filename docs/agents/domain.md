# Domain Docs

Use a single-context layout:

- Domain glossary: `CONTEXT.md`
- Operational source of truth: `AGENTS.md`
- Architecture decisions: create ADRs under `docs/adr/` only when a durable, hard-to-reverse decision needs to be recorded
- Historical implementation plans: `docs/plans/`, `docs/specs/`, `docs/reviews/`, and `docs/superpowers/`

`CONTEXT.md` should stay compact and domain-focused. Do not copy the full operational history from `AGENTS.md` into it.
